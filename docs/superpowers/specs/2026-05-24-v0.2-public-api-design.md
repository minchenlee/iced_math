# iced_math v0.2.0 — Public API Hardening Design

**Date:** 2026-05-24
**Status:** Approved
**Bump:** v0.1.1 → v0.2.0 (breaking: module visibility changes)

## Motivation

Pre-publish external review flagged the crate as a "good foundation, not fully
polished for outside users." Seven items, all to be addressed before first
crates.io publish:

1. Hide internal modules (`boxer`, `font`, `ir`, `parse`, `spacing`, `svg`) —
   currently `pub`, so users can depend on them and block future refactors.
2. Add a lower-level non-Iced rendering API (raw SVG bytes).
3. Drop forced Iced features (`x11`, `thread-pool`) from library deps.
4. Improve README quickstart (full minimal Iced view example).
5. Add rustdoc examples on public functions.
6. Clarify the stable public API surface (pre-1.0 note).
7. Resolve `.gitignore` / `Cargo.lock` inconsistency (lib should not track it).

## Public API (after)

```rust
// Iced widget entry points (unchanged signatures, reimplemented on Renderer)
pub fn inline<...>(src: &str) -> Element<...>;
pub fn block<...>(src: &str) -> Element<...>;

// New: lower-level builder (Option C — full config struct)
// Named MathRenderer (not Renderer) to avoid colliding with iced::Renderer
// and the generic `Renderer` type param in inline/block — keeps rustdoc/examples clear.
pub struct MathRenderer { /* font_size, display, color */ }
impl MathRenderer {
    pub fn new() -> Self;                      // 16.0 px, inline, black
    pub fn font_size(self, px: f32) -> Self;   // stored as-is; validated in to_svg
    pub fn display(self, yes: bool) -> Self;   // false = text-style, true = display
    pub fn color(self, c: Color) -> Self;
    pub fn to_svg(&self, src: &str) -> Result<Vec<u8>, Error>;
}
impl Default for MathRenderer { /* = new() */ }

// New public color type (keeps low-level path iced-free)
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct Color { pub r: u8, pub g: u8, pub b: u8 }
impl Color { pub const BLACK: Color; pub fn rgb(r,g,b) -> Color; }

// New public error type
#[derive(Debug)]
pub enum Error {
    Parse(String),
    /// font_size was not a finite, strictly-positive number.
    InvalidFontSize(f32),
}
impl std::fmt::Display for Error { ... }
impl std::error::Error for Error { ... }
```

All other modules become private (`mod`, not `pub mod`). `ir::Style` stays
internal; the builder maps `display: bool` → `ir::Style::{Display,Text}`. The
generic widget-renderer param in `inline`/`block` keeps the name `Renderer`
(no longer collides — the struct is `MathRenderer`).

## font_size validation

`font_size` is stored verbatim by the setter (builders stay infallible). `to_svg`
validates first: if `!font_size.is_finite() || font_size <= 0.0`, it returns
`Err(Error::InvalidFontSize(font_size))` before touching the pipeline. This keeps
a stable public API from ever emitting SVG with NaN/negative/zero dimensions or
transforms. `inline`/`block` always use the 16px default, so they cannot hit this.

## Color threading (wrap-only-when-non-default)

`svg::emit` currently emits `<path>`/`<rect>` with **no** `fill` attribute, so
they default to black. To support `.color()` **without churning the existing
snapshots**, `emit` takes a `Color` and:

- **`Color::BLACK` (default):** emit exactly as today — no `fill` attribute,
  no wrapper. Byte-for-byte identical to v0.1.1 output → snapshots unchanged.
- **non-default color:** wrap all children in a single `<g fill="#rrggbb">…</g>`
  group. SVG fill inheritance (honored by usvg/resvg and the iced svg widget)
  colors every glyph/rule with one attribute, instead of stamping each element.

This avoids per-element fill attrs entirely and means the default path produces
the same bytes the corpus snapshots already capture — **no insta re-accept**.

`inline`/`block` keep the black default (their signatures take no color); only
`MathRenderer` exposes `.color()`.

## Internal-test strategy

Privatizing modules breaks the ~19 integration test files in `tests/` that
import `iced_math::{boxer, ir, parse, font, svg, spacing}` — integration tests
can only see the **public** API.

Decision: **move internal tests into `src/` as `#[cfg(test)] mod tests`.**

- Pure-internal tests (boxer geometry, parse IR shape, font metrics, spacing,
  svg emit smoke) → relocate into the matching `src/<module>.rs`.
- Snapshot tests (`svg_snapshots`, `pixel_snapshots`) → **stay in `tests/`** but
  rewrite to drive the new public `MathRenderer::to_svg` API instead of internal
  `parse`/`boxer`/`svg`. They only need SVG bytes, which `to_svg` returns. The
  insta `glob!` corpus paths and snapshot dirs are unaffected.
- `tests/api.rs` → already public-only; unchanged.

Snapshot **values do not change**: the corpus renders at the black default,
which (per the wrap-only-when-non-default color scheme above) emits byte-for-byte
the same SVG as v0.1.1. **No insta re-accept expected.** If any snapshot diff
appears, it is a bug in this refactor — investigate, don't blindly accept.

## Cargo.toml

```toml
[dependencies]
iced = { version = "0.14", default-features = false, features = ["svg", "advanced"] }
# dropped: "thread-pool", "x11" — these were forced on every downstream user
# but are only needed to RUN a windowed app, which the library never does.
pulldown-latex = "0.7"
ttf-parser = "0.25"

[dev-dependencies]
insta = { version = "1", features = ["yaml", "glob"] }
resvg = "0.46"
tiny-skia = "0.11"
criterion = "0.5"
# The viewer example needs a windowing shell + executor + renderer. These used
# to ride in transitively via the library's "thread-pool"/"x11" features; now
# that the library no longer forces them, the EXAMPLE must request them here.
iced = { version = "0.14", default-features = false, features = ["wgpu", "tiny-skia", "thread-pool", "x11"] }
```

Library builds with only `svg` + `advanced`. `inline`/`block` need
`iced::advanced::{svg,text}::Renderer` traits + `widget::{svg,text,container}`
— all under `advanced`/`svg`, not windowing.

**Key risk (raised in review):** dropping `thread-pool`/`x11` from `[dependencies]`
removes them from the merged feature graph; the viewer example previously relied
on them transitively. The dev `iced` line above re-adds them so the example still
builds and runs — especially on Linux where `x11` matters. Both `cargo check
--example viewer` AND `cargo run --example viewer` are success criteria.

## Success criteria

- `cargo build --lib` succeeds with only `svg`+`advanced` library features.
- `cargo test` green (unit + snapshot), **no snapshot re-accept**.
- `cargo test --doc` passes (doctests on `inline`, `block`, `MathRenderer`).
- `cargo doc --no-deps` builds.
- `cargo check --example viewer` AND `cargo run --example viewer` both work.
- `to_svg` with non-finite/≤0 font_size returns `Err(Error::InvalidFontSize)`.
- Non-default color produces a `<g fill="#rrggbb">` wrapper; default color produces
  byte-identical output to v0.1.1 (no wrapper, no fill attrs).
- Only `inline`, `block`, `MathRenderer`, `Color`, `Error` reachable from outside.
- `cargo publish --dry-run` clean.
