# iced_math v0.2.0 — Public API Hardening Design

**Date:** 2026-05-24
**Status:** Approved
**Bump:** v0.1.1 → v0.2.0 (breaking: module visibility changes)

## Motivation

Pre-publish external review flagged the crate as a "good foundation, not fully
polished for outside users." Seven items, all to be addressed before first
crates.io publish:

1. Hide internal modules (`boxer`, `font`, `ir`, `parse`, `spacing`, `svg`) —
   currently `pub`, so users can depend on them and block future refactors.
2. Add a lower-level non-Iced rendering API (raw SVG bytes).
3. Drop forced Iced features (`x11`, `thread-pool`) from library deps.
4. Improve README quickstart (full minimal Iced view example).
5. Add rustdoc examples on public functions.
6. Clarify the stable public API surface (pre-1.0 note).
7. Resolve `.gitignore` / `Cargo.lock` inconsistency (lib should not track it).

## Public API (after)

```rust
// Iced widget entry points (unchanged signatures, reimplemented on Renderer)
pub fn inline<...>(src: &str) -> Element<...>;
pub fn block<...>(src: &str) -> Element<...>;

// New: lower-level builder (Option C — full config struct)
pub struct Renderer { /* font_size, display, color */ }
impl Renderer {
    pub fn new() -> Self;                      // 16.0 px, inline, black
    pub fn font_size(self, px: f32) -> Self;
    pub fn display(self, yes: bool) -> Self;   // false = text-style, true = display
    pub fn color(self, c: Color) -> Self;
    pub fn to_svg(&self, src: &str) -> Result<Vec<u8>, Error>;
}
impl Default for Renderer { /* = new() */ }

// New public color type (keeps low-level path iced-free)
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct Color { pub r: u8, pub g: u8, pub b: u8 }
impl Color { pub const BLACK: Color; pub fn rgb(r,g,b) -> Color; }

// New public error type
#[derive(Debug)]
pub enum Error { Parse(String) }
impl std::fmt::Display for Error { ... }
impl std::error::Error for Error { ... }
```

All other modules become private (`mod`, not `pub mod`). `ir::Style` stays
internal; the builder maps `display: bool` → `ir::Style::{Display,Text}`.

## Color threading

`svg::emit` currently emits `<path>`/`<rect>` with **no** `fill` attribute
(defaults to black). To support `.color()`, `emit` gains a fill parameter and
writes `fill="#rrggbb"` on every emitted element. Because the Iced widget path
rasterizes those same SVG bytes, color flows through `inline`/`block` for free
once the builder exposes it — but v0.2 keeps `inline`/`block` at black (their
signatures take no color); only `Renderer` exposes `.color()`.

## Internal-test strategy

Privatizing modules breaks the ~19 integration test files in `tests/` that
import `iced_math::{boxer, ir, parse, font, svg, spacing}` — integration tests
can only see the **public** API.

Decision: **move internal tests into `src/` as `#[cfg(test)] mod tests`.**

- Pure-internal tests (boxer geometry, parse IR shape, font metrics, spacing,
  svg emit smoke) → relocate into the matching `src/<module>.rs`.
- Snapshot tests (`svg_snapshots`, `pixel_snapshots`) → **stay in `tests/`** but
  rewrite to drive the new public `Renderer::to_svg` API instead of internal
  `parse`/`boxer`/`svg`. They only need SVG bytes, which `to_svg` returns. The
  insta `glob!` corpus paths and snapshot dirs are unaffected.
- `tests/api.rs` → already public-only; unchanged.

Snapshot **values do not change** (same pipeline, same 16px, black fill which
matches the current no-fill default → identical bytes), so no insta re-accept
is expected. (Verified during execution: if the explicit `fill="#000000"`
differs from the prior no-attr output, accept once and note it.)

## Cargo.toml

```toml
[dependencies]
iced = { version = "0.14", default-features = false, features = ["svg", "advanced"] }
# dropped: "thread-pool", "x11"
pulldown-latex = "0.7"
ttf-parser = "0.25"

[dev-dependencies]
# unchanged: insta, resvg, tiny-skia, criterion + iced wgpu/tiny-skia for viewer
```

Library must build with only `svg` + `advanced`. `inline`/`block` need
`iced::advanced::{svg,text}::Renderer` traits + `widget::{svg,text,container}`
— all under `advanced`/`svg`, not windowing. Verified at execution by
`cargo build` (no dev-deps).

## Success criteria

- `cargo build` (no features beyond defaults) succeeds with only `svg`+`advanced`.
- `cargo test` green (unit + snapshot).
- `cargo doc --no-deps` builds with doctests passing (`cargo test --doc`).
- Only `inline`, `block`, `Renderer`, `Color`, `Error` reachable from outside.
- `cargo publish --dry-run` clean.
